However, it is hosted in on the Git Hub Pages platform. The advances of following this procedure are:

  • Git Hub Pages offers a free hosting and allows also a domain redirection (this means that if you have registered a domain you can show the Site with your domain name).
  • The Git Hub Pages follow the rules of the Git Hub repositories, so it is wonderful for code sharing and maintenance.
  • The Git Hub Pages show only static Web Site, so there is low problem with code hacking and your Web Site is safe for virus attack.
  • The use of WordPress makes the aesthetic in standard level and ready to move the Web Site easily to a hosting provider with more functionality.

It could be the perfect solution if you want to show some post, diary pages results in a one way direction (from you to the others). This limit is given by the fact you have a very restricted use of the Git Hub server. You cannot, for instance get information back (from the viewers to you) and you cannot run any simulation. For this kind of activities you need a wider access to a server, and usually this is almost never free.

To achieve this solution you have to:

  1. Since you do not have a server for you, you have to build a new one in your local machine. This is done by a Virtualization Software that simulate the missing host provider.
  2. On this virtual server you can install WordPress, in a manner quite similar to a standard server installation (https://polgia0.github.io/2020/05/12/ciao-mondo/).
  3. Then you can build you Web Site using the WordPress statements.
  4. When finish, you have to make your Web Site static.
  5. You need an account on Git Hub Pages and achieve your personal repository (https://polgia0.github.io/2020/05/16/build-a-github-repository/).
  6. Then you have to move your Web Site in static form from your Local PC to the Git Hub Page repository.
  7. Finally, you can redirect a Web Domain (registered in and external hosting provider) to you Git Hub Pages repository and access to your Web Site with your most preferred domain name. This part is not compulsory and is not free because domain registration has a cost.

It looks difficults because many steps are required, but actually it is’not. Many step are very trivial operation like make aregistration or move a directory. In any case, nothing is really free and requires at least some work.

]]> Set up WordPress locally https://polgia0.github.io/2020/05/12/ciao-mondo/ Tue, 12 May 2020 11:24:05 +0000 https://polgia0.github.io/?p=1 [...] Read More]]> You can find herebelow detailed steps for the Web Site construction:

1. Build your local server

XAMPP is an Apache/MySQL resident package. It must be installed in the proposed directory C:\XAMPP, after is downloaded from: https://www.apachefriends.org/it/download.html

2. Get your WordPress copy

WordPress is a Content Management System (CMS) packaged in a .zip file that you can get from: https://it.wordpress.org/download/.

After decompression move the whole Worpress directory to the : C:\XAMPP\htdocs directory.

After the WordPress directory is moved, rename it, with the name of your Web Site. In my case I have renamed it: polgia0. In case you want to make more than one web site you have to copy the original WordPress directory many times and rename each copy with a different name, each for any web site you want to create.

3. Start your virtual server

Start XAMPP clicking on its Control Panel:

Now, push the Start Buttons for Apache and MySQL in the Application window. If everything goes well you should see in the log area positive feedback messages.

NOTE: I have noticed that XAMPP conflicts with a MySQL server already running. In this case you have first to stop the existing MySQL service with the local icons on the botton-right of the status bar:

Then you can start XAMPP correctly. When you have finished managing your Web Site you can restart the MySQL server again with the same tool.

4. Create a Database on MySQL

Now, open your browser (MS-Edge, Chome, etc.) and go to the address:

http://localhost/phpmyadmin/

This is a connection to a local virtual server installed on your PC by XAMPP and it performs like a server of a hosting provider. You should get a page like this:

By pressing the on Database you can easily create a new database by filling the name and pushing the Create button. The name of the database must be the name of your Web Site, i.e. the name that you have used at point 3 when you renamed the WordPress directory. In my case is again: polgia0.

5. WordPress installation in the local server

At this point you have a local sever (or hosting), a MySQL database and a copy of WordPress. However, you have to make aware all these elements about the existence of the others. Let’s start to connect WordPress to the MySQL Database. From your browser go to:

htpp://localhost/name Web Site/wp-admin/install.php

in my case this would be: localhost/polgia0/wp-admin/install.php. After a couple of welcome pages you should come to the following:

Fill the Database Name with te name of your Web Site, i.e. the same name that you have used renaming the directory of WordPress and the Database name in MySQL (in my case is always: polgia0). In this way you make aware the copy of WordPress that, on your local server, all the files of this Web Site are located in the renamed directory and that all the information are stored in the equal name Database.

Username is the user of MySQL Database, so put : root and Password is the password of root that is null. Since my SQL is installed in a local server this is safe. Database Host is clearly the local host address of your virtual server.

6. WordPress Configuration

If everything goes well, you reach the last step of starting WordPress. The final formis the following:

Here you can put the title of your Web Site that could be different from the name of your Web Site. In my case for instance I choose to call it: Good Advice. Than you have to put a user name specific foe WordPress, in my case I used my first name and than digit a password (WordPress suggest one that is impossible to remember, so, you are encouraged to change it). Although you are making a local installation, WordPress is not aware of, so it forces you to fill the e-mail address. Write whatever you like since it will be never used.

7. Use WordPress

If you did everything correct, at this point you can connect to the local copy of your WordPress at the address:

http://localhost/name Web Site/wp-admin

With Username and Password as defined in point 6. In my case the address is: localhost/polgia0/wp-admin and I see the main page of WordPress configured for my new Web Site.

From now, you can start working on WordPress, like you do with an external hosting, and start t build your new Web Site. Finally the address of your Web Site locally is:

http://localhost/name Web Site

That in may case is: http:/localhost/polgia0. Have fun !
